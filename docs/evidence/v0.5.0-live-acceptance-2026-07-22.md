## Qualified managed-storage topology

The accepted schemas were inside a Unity Catalog `MANAGED_CATALOG` created with
Databricks Default Storage. Each customer-owned evidence schema had no
schema-level storage root or location and inherited the catalog's managed
storage. dbtobsb created four managed Delta tables, three views, and two managed
Unity Catalog Volumes in that schema. It did not use an external table, external
Volume, external location, or product-created schema.

An earlier attended attempt against a different catalog failed closed with
`DBTOBSB_BOOTSTRAP_TABLE_CREATE_STORAGE_UNAVAILABLE`. Readback showed that the
selected schema could not create the required managed table. The failed runtime
graph was removed with the launcher's state-aware cleanup, its checkpoint was
preserved, and no catalog or subscription setting was changed in place. The
accepted run started from a separate clean checkout and a new empty schema in the
Default Storage catalog. This repeats the v0.4 storage recovery and proves this
topology; it does not qualify catalogs without usable managed storage.
