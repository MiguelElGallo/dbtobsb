# P1.1 local artifact-pair evidence — 2026-07-15

This record covers the offline P1.1 inspector only. It is not a Databricks runtime capture and does not qualify the candidate support row.

## Outcome

The inspector validated the three checked-in synthetic fixtures deterministically:

| Fixture | Pair result | Native dbt result | Intended lesson |
| --- | --- | --- | --- |
| `valid_success` | `PAIR_VALID` | `success=1` | Valid evidence and successful dbt outcome are separate facts. |
| `valid_dbt_failure` | `PAIR_VALID` | `error=1` | A valid evidence pair does not mean dbt succeeded. |
| `invalid_invocation_mismatch` | `PAIR_INVALID` | not interpreted | Files from different invocations cannot be trusted as one pair. |

Every fixture is generated by `capture/scripts/generate_artifact_pair_fixtures.py`, contains leak canaries, has a checksum record, and sets `runtime_evidence=false`. Run-results are synthetic. The source manifest came from a local `dbt parse`; neither that parse nor the sanitized fixtures prove Azure Databricks runtime behavior.

## Candidate compatibility evidence

The local fixture-source parse used dbt Core 1.11.12, dbt-databricks 1.12.2, dbt-common 1.37.5, dbt-adapters 1.23.0, dbt-protos 1.0.541, and dbt-spark 1.10.3. The raw parsed manifest contained 761 macros and produced zero validation errors against the exact Core 1.11.12 tagged manifest schema.

| Vendored schema | SHA-256 |
| --- | --- |
| Core 1.11.12 manifest v12 | `b290bb419026be3fa361ead7f0d8e3bb0eb6bca8c3a3fb7971f5d252fd0825b3` |
| run-results v6 | `1783bda55656bde624ed67640375640a2919a8608e2505e3a02c04cd139e2cdf` |

On 2026-07-15, the generic manifest-v12 URL served bytes identical to Core 1.11.7 and did not include the later tagged changes used by Core 1.11.12. P1.1 therefore vendors and verifies the exact tag bytes while still requiring the standard v12 URL in artifact metadata. The `dbt-adapters==1.24.5` path remains a separate unqualified candidate because its JavaScript macro-language value is outside that tagged schema.

The source parse ran on local CPython 3.12.13 for fixture preparation. The planned Databricks candidate remains Python 3.12.3 and requires a real locked-runtime test before qualification.
