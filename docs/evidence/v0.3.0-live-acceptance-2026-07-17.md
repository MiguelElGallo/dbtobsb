## Qualified release path

It selected an existing administrator-owned evidence schema and an existing observed-principal-owned dbt target schema. The observed Job and the collector/reconciler Jobs ran as distinct service principals. The read-only App received one SQL warehouse binding and `SELECT` bindings for only the three sanitized health views.

The finalized runtime used Databricks serverless environment client 5, Python `3.12.3`, dbt Core `1.11.12`, dbt-databricks `1.12.2`, Databricks SDK `0.117.0`, and Databricks SQL Connector `4.3.0`. The non-demo qualification project used the sealed `weather_release` selector and this graph:

```text

```

Installation created seven relational objects and two managed Volumes, applied the fixed runtime grants, deployed the three Jobs, deployed and verified the App, stopped the App, and left the reconciler schedule paused.
