## Qualified managed-storage topology

The live schemas were inside a Unity Catalog `MANAGED_CATALOG` with a
catalog-level managed storage root and location. Each customer-owned evidence
schema had no schema-level storage root or location and inherited the catalog's
managed storage. dbtobsb created four managed Delta tables, three views, and two
managed Unity Catalog Volumes in that schema. It did not use an external table,
external Volume, external location, or product-created schema.

This is the exact storage topology qualified by this packet. Another topology,
including a schema-level managed location or external storage, must repeat the
applicable release gates before it is described as qualified.
